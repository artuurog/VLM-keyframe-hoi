Learning Long-Horizon Robotic Manipulation from Human Videos via Interaction-Aware Keyframe Selection and Vision-Language Planning

Arturo Giacobbe1, Loris Roveda1,2, Francesco Braghin1
1Politecnico di Milano, Milan, Italy  ·  2IDSIA USI SUPSI, Lugano, Switzerland
arXiv Code Data

Abstract

Human videos are a promising way of collecting simple yet information-dense demonstrations for robotic manipulation. Vision-Language Models have recently emerged as an effective way to interpret long videos and image sequences, but this process is lengthy and produces low quality results that are not applicable to robotic manipulation.

In this work, we propose a novel methodology for task learning from long-horizon human video demonstrations. The proposed hand-object interaction identifier is robust to partial hand occlusions and can infer interactions even when hand keypoints are not visible or the hand moves erratically. A subset of keyframes is extracted from the video demonstration based on the minimization of a cost function.

A Vision-Language Model is then used to learn a structured PDDL task plan to be used during test time. We demonstrate that our approach outperforms state-of-the-art baselines both in simulation and on a real robot arm. We evaluate the proposed methodology on an unmanned robotic platform and observe significant improvements over state-of-the-art baselines in sorting and block-stacking tasks. Our experiments highlight the importance of incorporating interaction force feedback during task execution.

Approach

Method

Human Video Demonstrations

Long-horizon video demonstrations are recorded as raw human videos using standard RGB cameras. No robot teleoperation or specialized equipment is required.

Cost function evaluation

Hand motion and object tracking are extracted from the video, and a cost function is evaluated at each frame. The cost function takes into account several aspects of relative hand-object motion.

Hand-object interaction probability

Hand-object interaction probabilities are computed from the cost function by normalizing the values using a softmax operation.

Keyframe extraction

The video frames with the highest hand-object interaction probabilities are extracted and used for VLM reasoning.

Vision-Language Task Planning

The selected keyframes are fed to a VLM together with structured prompts. The VLM produces a PDDL task plan representing the high-level symbolic structure of the task, which is later used to guide robot execution.

Robot Execution

At test time, the robot follows the learned PDDL plan. The task pan is adapted to the specific scene thanks to object localization and VLM reasoning.

VLM Prompts

The following prompts are used to query the Vision-Language Model at each stage of the pipeline. Keyframe images are provided as visual context alongside each prompt.

Prompt 1 · Scene Understanding 
You are analyzing a sequence of keyframes from a human demonstration of a manipulation task.
Describe the objects present in the scene and their spatial relationships.
Focus on objects that are being actively manipulated by the human hand.
Prompt 2 · Action Grounding 
Given the keyframes and the scene description, identify the sequence of atomic actions performed.
For each action, specify: action type (grasp / place / push / …), object involved, and pre/post conditions.
Return the result as a structured PDDL action sequence.
Prompt 3 · PDDL Plan Generation 
Based on the identified action sequence, generate a complete PDDL problem file.
Include: objects, init state, goal state, and the ordered action plan.
The output must be syntactically valid PDDL and directly executable by a task planner.
